Расчеты значений частот для HUS

1. Для издания ЦАПом HUS звука требуемой частоты t_freq (target frequency) сэмплом, оцифрованным
с частотой d_freq (digitized frequency) при частоте смешения HUS mix_greq (mixing frequency) необходимо
в регистр "дробного шага" записать значение fract_step, расчитанное по формуле:

fract_step = d_freq/mix_freq*t_freq/523.251

Частота оцифровки сэмплов в ProTracker = 16726Гц.
Частота смешения HUS (по умолчанию)    = 48000Гц.

При таких условиях, выбранных для мод-плеера, формула вычисления дробного шага следующая:

fract_step = t_freq/1501,6171230419705847184024871458

2. Музыкальный модуль содержит в паттернах не номера нот и не частоты, которые нужно проиграть, а значение периода period, 
которое нужно записать в регистр звукового чипа Paula, чтобы получить частоту, соответствующую ноте.
Значение выходной частоты по периоду (для ProTracker) расчитывается по формуле:
t_freq = 7159090.5/32/period.
Соответственно, расчет значения дробного шага по периоду следующий:
fract_step = d_freq/mix_freq*7159090.5/32/period/523.251.
Для вышеуказанных параметров:
fract_step = 148,98709843677436832418858253496/period

3. Amiga ProTracker расчитан на проигрывание полутонов в трёх октавах: 1-я (Ля = 440Гц), 2-я (Ля = 880Гц)
и 3-я (Ля = 1760Гц). Однако, имеются плееры, поддерживающие большее число октав: Малая октава (Ля = 220Гц)
и 4-я (Ля = 3520Гц). Поддержим это расширение. Таким образом, минимальный period = 102, максимальный - 1814.
Эти данные нужны для определения разрядности вычислений с фиксированной точкой, чтобы избежать потери точности
при делении на максимальный период.
1814 - занимает 11 бит. Дробная часть fract_step - 12. Т.о. делимое в формуле вычисления дробного шага должно занимать
8 (целая часть) + 23 (дробная часть) = 31 бит.

Чтобы перевести десятичное делимое-дробь в шестнадцатиричное представление 8.23, нужно его разделить на 1/2^23 =
= 0x4A.7E593DD7, с округлением до целых - 0x4A.7E593DD8 в формате 8.23 бита. Или в формате 8.24 бита - 0x94.FCB27C.
(не определился - стоит ли экономить вычисления на 1 бите).
Итак: fract_step = 0x4A.7E593D/period.

Упростим вычисления для делителей сообразно их разрядности:
0x4A.7E593D   для делителя до 2048
0x25.3F2C9E   для делителя до 1024
0x12.9F964F   для делителя до  512
0x09.4FCB27   для делителя до  256
0x04.A7E593   для делителя до  128
